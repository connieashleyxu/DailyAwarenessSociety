Want to Get Into Coding? Here's How.

Industry

09/27/20

In our society, technology is so deeply rooted in everything we do, and code is the center of it all. Coding opens an avenue of opportunities for you to start making lasting impacts in the tech field one line at a time.

Whether you are a liberal arts major in college, a bored high school student, an adult just looking to learn a new skill, or anyone in between, coding is something you should highly consider learning. Coding improves your communication and collaborative skills in addition to teaching logic. [1] As Steve Jobs once said,

"I think everybody in this country should learn how to program a computer, because it teaches you how to think. I view computer science as a liberal art."

But how do you learn one of the most relevant languages in our world today? Below is a step-by-step guide on the steps to take to learn to code; the comprehensive guide links numerous FREE resources throughout the process as well as pertinent information about disparities in the tech/STEM field.


1) Make sure you have the right equipment, which is essentially just a functioning laptop (or monitor/desktop). If you want to get serious about coding, it may help to upgrade your computer.

For portability purposes, it may be a good idea to invest in a laptop rather than a desktop/monitor. Check out this link to find the best laptops for coding.

Mac or PC?
I’ve used both PC and Mac, and I liked Mac better (completely personal preference). The Mac was definitely more expensive, but there are some applications that are just for macOS, and you are able to download a Windows bootleg on your Mac, but cannot download macOS onto your Windows computer.


2) Before you jump into code, familiarize yourself with coding terminology.

Check out the link for a list of programming terms for beginners. As you read through the resource, it would be helpful to jot down basic notes of these terns so that you can refer back to them as needed.


3) Learn about the different coding languages because there are so many, and each one serves a different purpose. It’s important to be able to identify which language you want to learn. Below is a shortlist of some of the most popular languages:

  • HTML: Used for web browser contents (front-end)
  • CSS: Used for design web browser contents (front-end)
  • Java/JavaScript: Used a lot in web development (object-oriented programming)
  • C/C++/C#: Object-oriented programming
  • Swift/Kotlin: Swift is for iOS; Kotlin is for Android (both are object-oriented programming for app development)
  • Ruby: Object-oriented programming (back-end); Ruby is easier to learn because it’s closer to the English language as a higher-level language
  • Python: Used a lot in machine learning/artificial intelligence (higher-level language and back end)
  • There are a lot more coding languages

Check out this source for a more in-depth list of what each language does.

Check out this source for a list of every coding language there is; this list will only continue to grow.


4) Learn the basics of code from free videos on YouTube.

No cap, I first learned to code by watching this playlist about Introduction to Computer Science and Programming. It’s really inspiring knowing that you are learning from professors in great universities (without having to pay the hefty tuition price).

Here are some more playlists that you can check out.

  • Introduction to Algorithms
  • Introduction to Computer Science and Programming in Python


5) Start coding with MIT's Scratch.

Scratch uses block-based programming that makes the logic behind code easy to visualize. It’s super important to get into that “logic” mindset when learning to code as this is the one thing that you will need in every coding language.

Follow along with this free YouTube playlist to learn Scratch.

You can also practice with instructions from Scratch itself once you feel more comfortable navigating your first steps into the coding realm.


6) Get yourself familiarized with coding environments and coding formats (outside of block code). Here are some popular coding environments:

  • Eclipse (download)
  • Xcode (download; only for Mac)
  • Glitch (online and collaborative)
  • Repl.it (online and collaborative)
  • CodePen (online)
  • Terminal/Command Shell (boot camps that come automatically with your computer)
  • Here is a list of more Integrated Development Environments (IDE)

Which IDE should I use?
It’s totally up to your personal preference to choose which environment you want to work in. Start by picking one that you enjoy the interface of!

Here is a list of some popular resources used in the coding community that you will want to get acquainted with:

  • GitHub (get an account and learn push/pull commands to save your code in repositories)
  • Stack Overflow (a great place to go for troubleshooting)
  • W3 Schools (a great place to go for coding basics)
  • Google (you just need to ask the right questions, and you will find literally anything; Google will be your BFF in this process)

How do I format my code?
It’s imperative to format and comment your code because you are typically going to be working in a team environment in the industry, and others will need to be able to quickly understand your code.

It’s important to build good habits of neatly formatting code from the start of your coding journey. Check out this resource for a guide on code formatting.

You will also want to learn how to comment on the different coding languages you want to learn.

In terms of getting used to coding formats, you can always inspect the code of your favorite websites by typing in the following keyboard command while on any website:

(for PC) ctrl shift i
(for Mac) command option i

You should be able to see the code of the website where you can reverse engineer the given code.


7) Start learning a coding language. Definitely easier said than done, but here are some helpful tips on where to start.

For self-studiers:
If you are able to self-study and want a more in-depth experience, you can also pick up a book about coding.* I took this route when learning how to code in Python. But also note that at this point, I had already learned to code in many languages and had ample experience.

For non-self-studiers:
For many other people, including myself, it may be difficult to learn something completely foreign without a little bit of guidance and structure. So, I highly recommend attending a coding camp and/or workshop.* This way, there is a curriculum you can follow, and you will have the help of instructors and peers throughout the program.

You can also enroll in an online class, as it provides a similar level of structure.*

  • If you are interested in web development, start with learning HTML and CSS as these are the easiest of the languages to learn.

    I recommend coding in Glitch, CodePen, or Repl.it for your IDEs because they are all fully online and automatically saves your work as your code.

    To actually learn HTML/CSS, Code Academy offers courses for learning HTML and CSS. W3 Schools is another resource that allows you to learn HTML.

  • Next, you can start learning an object-oriented language like JavaScript or C++.

    I recommend coding in Atom (only for Mac), Xcode (only for Mac), Terminal/Command Shell, or Notepad++.

    Or, if you are coding for app development, I recommend using Xcode (for iOS) or Android Studio (for Android).

    Once you learn one object-oriented coding language, you basically know all of them -- it’s just a matter of learning different syntax.

*At the bottom of the article, I have linked resources on where to go for camps, workshops, online classes, and books.


8) If you want to get into machine learning, it would be wise to learn Python and Java. Java is incredibly similar to JavaScript, so learning JavaScript in the previous step would help tremendously.

To learn Python, I recommend taking the Google Machine Learning Crash Course to get the basics of Python as it relates to machine learning, which pairs with the Google Colab IDE.


9) Keep practicing and learning. Here are some parting tips that are important to internalize as you embark on your learning adventures with code:

  • The learning process takes time and PRACTICE. Don’t expect to be a coding genius in a week; if it were that easy, there wouldn’t be such a high demand for coders. Understand that this is a time commitment. Just like with sports/playing instruments, you can’t get really good until you’ve practiced as the skills don’t come overnight. So, don’t give up!!
  • Don’t rush into things, take your time, and enjoy the process.
  • Learning from your mistakes is the best way to learn; and trust me, there will be A LOT of mistakes.
  • The fastest and most efficient way to learn code, in my opinion, is to go to a coding boot camp because it’ll hold you accountable and ensure that you have somewhere to go with your questions.
  • GOOGLE IS YOUR BEST FRIEND. If you have the right questions, you will find just about everything you are looking for. It would also be helpful to learn how to read the formatting of Stack Overflow.
  • There really isn’t a moment where “you know everything there is to know” in code because there’s always more to know as new things are developing every day; so, you will constantly be learning.
  • It takes time to find what you like to do. Coding is a big world, and you might only like a part of it (or all of it) and both are perfectly OK!


10) Be mindful of your impact. As you get into the tech and/or coding field, it’s important to stay cognizant of the disparities that happen in the field.

The gender gap (especially in the tech field) is still a huge problem today. Below are some resources to educate yourself on the gender gap:

This article from the Economy Policy Institute provides a definition of the gender pay gap and delves into the research and statistics behind the problem.

This article from Forbes gives great insight into the pay inequality for women in the tech field.

I also recommend watching “Code: Debugging the Gender Gap” on Amazon Prime; the documentary was released in 2015 and brings up many interesting points about women in STEM. While this documentary doesn’t address the disparities of womxn in STEM, it still brings great perspectives on what it’s like not being a male in the industry.

Brown’s WiSTEM said it best, the reason we say womxn as opposed to women is that “As micro-aggressions work to subtly enforce sexist ideals and tendencies among individuals, the word ‘women’ subtly enforces the concept that womxn are second to men. The orthography is intersectional and is used to stress visually the symposiums inclusive approach to gender and gender identity. The Symposium values diversity. We encourage womxn from historically under-represented groups, and femme, trans-womxn, and non-binary persons in STEM fields to apply.”

Next Steps:
So you’ve learned to code, but how do you make something with what you’ve learned?

1) Passion projects. Ever had an idea for a website, app, etc.? Now’s your time to try to create it. Passion projects are great for practice and you can build a portfolio from it!

Project ideas for beginners:

  • https://www.makeuseof.com/tag/beginner-programming-projects/
  • https://www.programmingforbeginnersbook.com/blog/what_should_i_make_beginner_programming_project_ideas/

2) Hackathons. There are different types of hackathons for all ages and experience levels. These are great because you usually get an opportunity to win money and other prizes.

  • Virtual upcoming hackathons
  • Find a hackathon near you
  • If you are attending university, your own university probably has hackathons going on. Reach out to the executives/officers of your coding club(s) to find out more information!
  • Google “hackathons near me”

3) See if any of your family/friends want anything related to code done. While you may not be able to charge as much in the beginning, it’s good practice to get into the groove of making something to someone else’s wanting. It may be helpful to start with the comfort of understanding friends/family.

Get started on learning a new skill today, and what better skill to learn than an industry-standard -- coding?
